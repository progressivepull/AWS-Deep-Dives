# 16.6 Using Elastic Beanstalk best practices 
 
- **Enhanced Health Monitoring**
  - Recommended to enable for deeper visibility into environment health
  - Provides more detailed metrics and diagnostics
  - This is a **paid Elastic Beanstalk feature**

- **Log Streaming**
  - Recommended to enable log streaming to **Amazon CloudWatch**
  - Allows centralized monitoring and troubleshooting
  - Incurs **additional CloudWatch charges**

- **Log Retention**
  - Recommended to retain logs even after environment termination
  - Retention period is configurable
  - Default retention is **7 days** once enabled

- **Managed Platform Updates**
  - Recommended to enable for automatic platform maintenance
  - Configure:
    - Maintenance window
    - Patch level (security, minor, etc.)

- **Instance Replacement Option**
  - Optional setting during managed updates
  - Terminates old instances and launches new ones with updated platform
  - Useful for applying updates cleanly

- **Major vs Minor Updates**
  - Minor updates can be handled by managed platform updates
  - Major version changes require:
    - Creating a new environment
    - Migrating or swapping environments instead of in-place upgrades

- **Key Takeaway**
  - Following these best practices improves reliability, observability, and maintainability
  - Be mindful of added costs when enabling enhanced monitoring and logging
