# 13.3 Exploring SNS transports 
 
 ## Amazon Simple Notification Service (SNS) Transport Methods â€” Summary

- Amazon SNS supports multiple message delivery (transport) methods.
- Supported subscriber types include:
  - AWS Lambda
  - Simple Queue Service (SQS)
  - HTTP and HTTPS webhooks
  - Email
  - SMS
  - Mobile push notifications

- **Mobile Push Notification Services supported by SNS**:
  - Amazon Device Messaging
  - Apple Push Notification Service
  - Baidu Cloud Push
  - Google Cloud Messaging
  - Microsoft Push Notification Service (legacy Windows Phone)
  - Windows Push Notification Service

- **Email notifications**:
  - Messages can be sent to email subscribers of an SNS topic.
  - Email subscriptions must be verified by the recipient before activation.

- **SMS notifications**:
  - SMS messages can be sent without subscription confirmation.
  - SMS is only supported in certain AWS regions.

- **Targeted marketing use cases**:
  - SNS can be used for marketing campaigns via email and SMS notifications.

- **Mobile push notification use cases**:
  - Commonly used for application notifications, such as informing users when an in-app event occurs.
  - Example: notifying mobile game users that their energy has been replenished.

- **Invoking serverless compute with Lambda**:
  - Lambda functions can subscribe directly to SNS topics.
  - Published messages become the payload (input) for the Lambda function.
  - Lambda can process data, store it in databases, forward it to queues, or invoke other AWS services.

- **Webhook subscriptions**:
  - SNS supports HTTP and HTTPS endpoints.
  - HTTPS provides encrypted transport using SSL certificates.
  - Messages are delivered using HTTP POST requests.
  - Supports Server Name Indication (SNI).
  - Basic authentication can be configured using `user:password@` in the endpoint URL.
